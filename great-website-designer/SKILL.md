# Great Website Designer

This skill provides a framework for designing and auditing websites based on industry-leading principles of usability, aesthetics, and technical performance.

## Core Design Principles

### 1. User-Centric Empathy
- **Goal:** Design for the user's needs, not the designer's ego.
- **Action:** Prioritize ease of use. Every element should serve a purpose in the user's journey.
- **Checklist:**
    - Is the primary action (CTA) obvious?
    - Can a user find contact information in under 5 seconds?
    - Is the navigation intuitive and consistent?

### 2. Visual Hierarchy & Literacy
- **Goal:** Guide the user's eye through the page in a logical order.
- **Principles:**
    - **Scale:** Larger elements attract more attention.
    - **Color & Contrast:** Use high contrast for important text and buttons.
    - **Typography:** Limit to 2-3 font families. Use a clear heading structure (H1 -> H2 -> H3).
    - **Whitespace:** Use generous padding and margins to prevent "visual noise" and improve readability.

### 3. Responsive & Mobile-First
- **Goal:** Ensure a seamless experience across all devices.
- **Action:** Design for the smallest screen first.
- **Checklist:**
    - Are touch targets (buttons/links) large enough (at least 44x44px)?
    - Does the layout break or overflow on narrow screens?
    - Is text readable without zooming?

### 4. Accessibility (A11y)
- **Goal:** Make the web inclusive for everyone.
- **Action:** Adhere to WCAG 2.1 standards.
- **Checklist:**
    - **Alt Text:** Do all images have descriptive alt attributes?
    - **Contrast:** Is the text-to-background contrast ratio at least 4.5:1 for normal text?
    - **Keyboard Nav:** Can the site be navigated using only the TAB key?
    - **Semantic HTML:** Use proper tags (`<nav>`, `<main>`, `<footer>`, `<header>`) instead of generic `<div>`s.

### 5. Performance & Speed
- **Goal:** Minimize load times to retain users and improve SEO.
- **Action:** Optimize assets and minimize code bloat.
- **Checklist:**
    - **Images:** Use modern formats (WebP/AVIF) and responsive sizes (srcset).
    - **Fonts:** Minimize the number of font weights loaded.
    - **Lazy Loading:** Use `loading="lazy"` for off-screen images.

## Audit Workflow

When auditing an existing website, follow these steps:

1. **Initial Impression:** Does the site look modern and trustworthy?
2. **Goal Check:** Is it clear what the business does and what the user should do next?
3. **Responsive Test:** Resize the window. Identify any layout issues.
4. **Accessibility Scan:** Check image alt tags and color contrast.
5. **Technical Check:** Review the HTML structure for semantic correctness.

## Design Patterns

- **Hero Section:** Clear H1, supportive sub-text, and a primary CTA.
- **Trust Signals:** Testimonials, logos, or certifications.
- **Contact:** Footer with address, phone, and hours.
- **Clear CTAs:** Use distinct colors for primary actions.
